Traversal Algorithms
BFS — Breadth First SearchTraversal
Explores all nodes level by level using a queue. Visits all nodes at distance k before nodes at distance k+1. Guarantees shortest path in unweighted graphs.
Time
O(V+E)
Space
O(V)
Data Struct
Queue
Works on: directed + undirected, weighted + unweighted
Use for: shortest path (unweighted), level-order traversal, finding connected components
DFS — Depth First SearchTraversal
Goes as deep as possible before backtracking. Back edges indicate cycles. Foundation for topological sort, SCC, bridges.
Time
O(V+E)
Space
O(V)
Data Struct
Stack/Rec
Works on: directed + undirected, weighted + unweighted
Use for: cycle detection, topological sort, SCC, path finding, maze solving
Cycle Detection
Cycle Detection — BFSCycle
Uses BFS with parent tracking. If we visit a node that's already visited and it's not the parent of current node — cycle found.
Time
O(V+E)
Space
O(V)
Data Struct
Queue+Map
Key: if vis[nbr]==true AND nbr != parent[node] → CYCLE
Cycle Detection — DFSCycle
Uses DFS recursion with parent parameter. If neighbor is visited and not parent → back edge → cycle exists.
Time
O(V+E)
Space
O(V)
Data Struct
Recursion
Key: if vis[nbr]==true AND nbr != parent → CYCLE (back edge)
Shortest Path Algorithms
Dijkstra's AlgorithmShortest Path
Greedy algorithm. Always picks the node with minimum distance. Uses a priority queue (min-heap or set). Single source shortest path.
Time
O(E log V)
Space
O(V)
Type
Greedy
CONSTRAINT: All edge weights must be ≥ 0. Negative weights break the greedy assumption.
Use for: GPS navigation, network routing, single-source shortest path with non-negative weights
Bellman-FordShortest Path
Relaxes all edges V-1 times. Can handle negative weights. If after V-1 passes we can still relax → negative cycle exists.
Time
O(V·E)
Space
O(V)
Type
DP
DETECTS negative cycles. If cycle exists — shortest path = undefined (-∞)
Use for: graphs with negative weights, currency arbitrage detection
Floyd-WarshallShortest Path
All-pairs shortest paths. Uses DP: dist[i][j] = min(dist[i][j], dist[i][k]+dist[k][j]) for each intermediate node k.
Time
O(V³)
Space
O(V²)
Type
DP
CONSTRAINT: No negative cycles. Handles negative weights but not cycles.
Use for: all-pairs shortest path, small dense graphs, transitive closure
DAG Shortest PathShortest Path
Uses topological order + relaxation. Faster than Dijkstra. Works with negative weights since DAG has no cycles.
Time
O(V+E)
Space
O(V)
Type
Topo+DP
CONSTRAINT: DAG only (Directed Acyclic Graph). No cycles allowed.
Use for: project scheduling, build dependency resolution
Topological Sort
Topological Sort — DFSTopo Sort
DFS-based. After visiting all neighbors of a node, push it to stack. Stack gives reverse topological order. Pop stack for final order.
Time
O(V+E)
Space
O(V)
Data Struct
Stack+DFS
CONSTRAINT: DAG only. If cycle exists → topological sort is impossible (undefined).
Kahn's Algorithm — BFSTopo Sort
BFS-based using in-degree. Start with nodes of in-degree 0. Process each, reduce neighbor in-degrees. If count processed < V → cycle exists.
Time
O(V+E)
Space
O(V)
Data Struct
Queue
DETECTS cycles: if processed_count < V → cycle exists → not a DAG.
SCC & Bridge Finding
Kosaraju's SCCSCC
Two DFS passes. Pass 1: DFS on original graph, push to stack in finish order. Pass 2: DFS on reversed graph in finish order — each DFS tree = one SCC.
Time
O(V+E)
Space
O(V)
Passes
2×DFS
CONSTRAINT: Directed graph only. Reverses all edges in Pass 2.
Use for: finding strongly connected components, web crawling, social network analysis
Tarjan's Bridge FindingBridges
Uses disc[] (discovery time) and low[] (lowest reachable). Bridge condition: low[v] > disc[u] means edge u→v is a bridge — removing it disconnects graph.
Time
O(V+E)
Space
O(V)
Arrays
disc+low
Works on undirected graphs. Bridge = critical connection in network.
Use for: network reliability, critical infrastructure, finding weak points
Minimum Spanning Tree
Prim's AlgorithmMST
Grows MST from a source. At each step picks the minimum weight edge connecting MST to a non-MST node. Uses priority queue.
Time
O(E log V)
Space
O(V)
Type
Greedy
CONSTRAINT: Undirected weighted graph. Better for dense graphs.
Use for: network design, cable laying, circuit design
Kruskal's AlgorithmMST
Sort all edges by weight. Add edge if it doesn't form a cycle (check using DSU/Union-Find). Continue until V-1 edges added.
Time
O(E log E)
Space
O(V)
Type
DSU+Sort
CONSTRAINT: Undirected weighted graph. Better for sparse graphs.
Use for: network design, clustering, image segmentation
